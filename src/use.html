How to use the Application

User identity

  • Provide the relevant User Identity details
  • Dont try to provide any invalid data to the form
  • Submit the form after verification of details.
  • METAMASK window will be opened and sign the transaction
  • If not interested to add details REJECT the transaction
Add User Identity

KYC document

  • Provide the relevant document details and select valid KYC document type from list.
  • Submit the form after verification of details.
  • After few seconds a alert message shown that 'File added to IPFS and returned the Hash'
  • METAMASK window will be opened and sign the transaction to add the hash to ethereum blockchain
Add KYC Document

Blockchain Ledger

  • Provide the valid block Number in the text box.
  • Select the type of reciept you want.
  • Block receipt shows block parameters
  • Transaction receipt shows transaction parameters
  • If transaction receipt shows nothing for a block then it is a dummy or empty transaction
Block Explorer Block Statistics

Request\Response History

  • It shows the "Request transactions" and the completed Reponse transactions.
  • Data or Document will be provided only if the owner of the identity grants permission to use.
  • After owner grants permission the "pending" request will be switch to "View Data" or "View Document".
  • If the owner rejects the request then the pending will be switch to "Rejected"
  • Request as well as the Response transactions for a particular account only displayed.
Request\Response

Organization

  • This project includes two major organizations with public addresses "Government" and "Bank".
  • Organizations can request the user for a identity data (Data Request form) or a document (KYC Request form) through respective forms.
  • Valid username is required for sending the request to the owner of the data or document.
  • METAMASK window will be opened and sign the transaction
  • If not interested to request the details REJECT the transaction
